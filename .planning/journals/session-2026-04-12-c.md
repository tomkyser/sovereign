## Objective

## Key Findings

### Zod `_._zod` Crash (FIXED in code, partially deployed)
- External tools with `inputJSONSchema` but no Zod `inputSchema` crash the CC execution pipeline
- `toolToAPISchema()` correctly checks `inputJSONSchema` first ‚Äî API schema works
- But **10+ callsites** call `.inputSchema.safeParse()` unconditionally without checking `inputJSONSchema`
- Files: `hooks.ts:1405`, `permissions.ts:606/1119/1215`, `toolOrchestration.ts:97`, `toolExecution.ts:615`, `StreamingToolExecutor.ts:104`, `permissionLogging.ts:50`, `extractMemories.ts:196`
- **Fix applied:** Added `TOOL_ZOD_SHIM_CODE` to loader ‚Äî borrows `inputSchema` from `_b[0]` for external tools
- **Problem with fix:** `_b[0]` is the Agent tool with a typed schema, not a passthrough. Need MCPTool's `z.object({}).passthrough()` instead.

### Binary Corruption ‚Äî Two Separate Causes
1. **Node.js v24 `fs` operations** corrupt Mach-O binaries. `fs.copyFile()`, and likely `fs.writeFile(binaryBuffer)`, replace non-UTF-8 bytes with U+FFFD (`ef bf bd`). 201MB clean ‚Üí 304MB corrupted. Shell `/bin/cp` is binary-safe.
2. **CC auto-updater** in sessions without `DISABLE_AUTOUPDATER=1` continuously overwrites the installed binary. Race window is sub-millisecond.

### Installer Bug
- `curl -fsSL https://claude.ai/install.sh | bash -s -- 2.1.101` produces corrupted binary
- Direct GCS download works: `curl -fsSL -o $TARGET "$GCS_BUCKET/2.1.101/darwin-arm64/claude"`
- GCS bucket: `https://storage.googleapis.com/claude-code-dist-86c565f3-f756-42ad-8dfa-d59b1c096819/claude-code-releases`

### Binary Vault (NEW architecture)
- Created `~/.claude-governance/binaries/virgin-2.1.101.bin` ‚Äî immutable, verified source of truth
- Flow: download ‚Üí verify magic ‚Üí lock ‚Üí copy to working path with `/bin/cp` ‚Üí apply ‚Üí deploy
- Apply workaround: pre-create `native-binary.backup` with `/bin/cp` so apply doesn't try `fs.copyFile()`

### Tool Validation Mismatch
- After Zod fix, Ping tool is visible to model (API schema works via `inputJSONSchema`)
- But `safeParse()` on the borrowed Agent schema rejects Ping's `{message}` input, expects `{description, prompt}`
- Need to borrow from MCPTool (`wj1()` = `z.object({}).passthrough()`) instead of `_b[0]` (Agent tool)

### Prompt Overrides Not Matching
- 8/8 prompt overrides fail verification on fresh 2.1.101 download
- The pieces-based matching pipeline may have version-sensitive hashes
- Needs investigation ‚Äî may be a prompt data file issue

## Decisions

- **Never use Node.js `fs` for binary file operations** ‚Äî always shell `/bin/cp` or `execSync`
- **Pre-create backup with `/bin/cp`** before running apply to avoid corruption
- **`~/.tweakcc/` deleted** ‚Äî no longer needed
