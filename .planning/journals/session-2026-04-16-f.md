# Session 2026-04-16-f — Repack Crash: Complete Root Cause & Fix Plan

## Executive Summary

The 27/27 SOVEREIGN patches are correct against the binary's JavaScript. The repack crash
has been fully diagnosed across two sessions (e, f). The root cause is a three-layer
incompatibility: Bun bytecode compilation → npm source is ESM → ESM can't be CJS-wrapped.
The fix is to use esbuild to transform the npm ESM source to CJS before embedding.

---

## THE PROBLEM

Claude Code 2.1.101's Bun binary stores `cli.js` as **bytecode-compiled** content.
The source text in the binary starts with `// @bun @bytecode @bun-cjs` and function
bodies are compiled stubs — not readable JavaScript. Our governance patches match
patterns in these stubs successfully, but the repacked binary crashes:

```
TypeError: Expected CommonJS module to have a function wrapper.
If you weren't messing around with Bun's internals, this is a bug in Bun
Bun v1.3.12 (macOS arm64)
```

---

## ROOT CAUSE (Three Layers)

### Layer 1: Bytecode Compilation
- Binary content prefix: `// @bun @bytecode @bun-cjs\n(function(exports, require, module, __filename, __dirname) {...})`
- Function bodies are bytecode stubs, not interpretable JS
- `encoding: 1` = bytecode mode; Bun uses bytecode blob (111MB) to execute
- Clearing bytecode + setting `encoding: 0` doesn't work because the source stubs aren't valid JS

### Layer 2: npm Source is ESM
- `@anthropic-ai/claude-code@2.1.101` has `"type": "module"` in package.json
- `cli.js` starts with: `import{createRequire as Ej5}from"node:module"`
- Contains: **753** static `import{} from`, **67** `import X from`, **12** `import.meta`, **6** top-level `await`
- The first static import is at character 307 of cli.js

### Layer 3: ESM ↔ CJS Incompatibility
- Bun binary module format requires a CJS function wrapper: `(function(exports, require, module, __filename, __dirname) {...})`
- ESM `import/export` statements are **top-level-only** JavaScript syntax
- They CANNOT appear inside a function body — the JS parser rejects them
- Without the CJS wrapper, Bun throws "Expected CommonJS module to have a function wrapper"
- Result: CJS wrapper = parse error; no wrapper = Bun loader error

### fetchNpmSource() — Add to patches/index.ts
```typescript
// Location: claude-governance/src/patches/index.ts (in Helper Functions section)
// Dependencies: import { execFileSync } from 'node:child_process' (add to imports)
// Uses existing: fsSync (import * as fsSync from 'node:fs'), path, os, debug

function fetchNpmSource(version: string): Buffer | null {
  const tmpDir = fsSync.mkdtempSync(path.join(os.tmpdir(), 'claude-gov-npm-'));
  try {
    debug(\`fetchNpmSource: Downloading @anthropic-ai/claude-code@\${version}\`);
    execFileSync('npm', ['pack', \`@anthropic-ai/claude-code@\${version}\`,
      '--pack-destination', tmpDir], { stdio: 'pipe', timeout: 30_000, cwd: tmpDir });
    
    const tgz = fsSync.readdirSync(tmpDir).find((f: string) => f.endsWith('.tgz'));
    if (!tgz) return null;
    
    execFileSync('tar', ['xzf', path.join(tmpDir, tgz), 'package/cli.js'],
      { stdio: 'pipe', timeout: 30_000, cwd: tmpDir });
    
    const cliJsPath = path.join(tmpDir, 'package', 'cli.js');
    if (!fsSync.existsSync(cliJsPath)) return null;
    
    return fsSync.readFileSync(cliJsPath);
  } catch (error) {
    debug(\`fetchNpmSource: Failed: \${error instanceof Error ? error.message : String(error)}\`);
    return null;
  } finally {
    try { fsSync.rmSync(tmpDir, { recursive: true, force: true }); } catch {}
  }
}
```

## THE FIX: ESM → CJS Transformation

### What Needs to Happen
1. Detect bytecode prefix in extracted binary content
2. Fetch clean ESM source from npm
3. **Transform ESM → CJS** (this is the missing piece)
4. Wrap transformed CJS in function wrapper
5. Apply governance patches
6. Repack with cleared bytecode + encoding=0

### ESM → CJS Transformation Options

#### Option A: esbuild (Recommended)
```bash
esbuild cli.js --bundle --format=cjs --platform=node --outfile=cli.cjs
```

- esbuild is available via npx (v0.28.0 on this machine)
- `--platform=node` keeps Node builtins external
- `--format=cjs` converts ESM → CJS syntax
- `--bundle` is needed for esbuild to resolve and transform imports
- Handles import.meta.url → `__filename` equivalent
- **Risk**: top-level `await` (6 occurrences) needs `--supported:top-level-await=true` or wrapping

Implementation in `fetchNpmSource`:
```typescript
// After extracting cli.js, before returning:
execFileSync('npx', ['esbuild', cliJsPath, '--bundle', '--format=cjs',
  '--platform=node', '--outfile=' + cjsPath],
  { stdio: 'pipe', timeout: 60_000, cwd: tmpDir });
return fsSync.readFileSync(cjsPath);
```

#### Option B: Regex Transform
Manual regex replacement of ESM syntax. More brittle but no external dependency:
- `import{X}from"Y"` → `const{X}=require("Y")`
- `import X from"Y"` → `const X=require("Y")`  
- `import.meta.url` → `require("url").pathToFileURL(__filename).href`
- Top-level `await` → wrap in async IIFE

**Risk**: 753+67 = 820 import statements, each with slightly different patterns.
Regex fragility with minified code.

### After Transformation
The CJS-transformed content gets:
1. Stripped of shebang (`#!/usr/bin/env node`)
2. Wrapped in CJS function: `(function(exports, require, module, __filename, __dirname) {\n` + content + `\n})`
3. Patched with governance changes (25/27 should apply, patterns match)
4. Repacked into binary with `clearBytecode=true`, `encoding=0`
5. Binary raw-overwritten (18MB new << 128MB original, fits in place)
6. Re-signed with ad-hoc codesign
